# claude-usage-plugin

یک افزونه مینیمال [Claude Code](https://claude.com/claude-code) با فقط یک وظیفه:
گزارش میزان مصرف حساب Claude شما در بازه‌ی **۵ ساعته** و **هفتگی** — همان اعداد
**رسمی** که خود Claude Code نمایش می‌دهد، نه یک تخمین.

این افزونه **هیچ محدودیتی اعمال نمی‌کند**. نه آستانه‌ای دارد، نه منطق مسدودسازی،
و هیچ‌چیز هرگز بلاک نمی‌شود. افزونه فقط می‌خواند و گزارش می‌دهد، از طریق دو ابزار
[MCP](https://modelcontextprotocol.io) بدون ورودی. بدون تماس شبکه، بدون کوکی
نشست، بدون اسکرپینگ — داده مستقیماً از خود Claude Code می‌آید.

## چرا این پروژه ساخته شد

خود Claude Code میزان دقیق مصرف محدودیت نرخ شما را می‌داند، اما فقط آن را روی نوار
وضعیت نقاشی می‌کند — هیچ راه داخلی‌ای برای مدل، یا خود شما، برای *پرس‌وجو* کردن آن
وجود ندارد. `claude-usage-plugin` برای مواقعی است که می‌خواهید وسط یک نشست بپرسید
«چقدر از بازه‌ی ۵ ساعته و هفتگی‌ام را مصرف کرده‌ام؟» و همان عدد رسمی نوار وضعیت را
بگیرید:

- درصد مصرف‌شده و زمان بازنشانی بازه‌ی **۵ ساعته**،
- درصد مصرف‌شده و زمان بازنشانی بازه‌ی **هفتگی** (۷ روزه)،

فقط با داده‌ای که Claude Code از قبل ارائه می‌دهد — بدون کلید API، بدون اسکرپینگ،
و بدون تخمین از روی شمارش توکن‌های محلی.

## چطور کار می‌کند

Claude Code (نسخه ۲.۱.۸۰ به بالا، Pro/Max) یک payload از نوع JSON شامل
`rate_limits.{five_hour,seven_day}` — که هرکدام یک `used_percentage` رسمی و یک
`resets_at` دارند — را در هر بار رندر نوار وضعیت به هر فرمانی که به‌عنوان هوک
**statusLine** ثبت شده باشد می‌فرستد. این افزونه آن را به یک نویسنده و یک خواننده
تقسیم می‌کند:

- **`capture-statusline.js`** — هوک statusLine. در هر رندر، `rate_limits` رسمی را
  در `~/.usage-guardian-mcp/statusline-latest.json` ذخیره می‌کند و نوار وضعیت را
  چاپ می‌کند (مثلاً `Opus 4.8 · 5h 86% · 7d 80%`). بدون شبکه، هرگز خطا پرتاب
  نمی‌کند، و هرگز نوار وضعیت شما را خالی نمی‌کند.
- **`index.js`** — سرور MCP. آن ذخیره‌شده را دوباره می‌خواند و به‌صورت دو ابزار
  بدون ورودی در دسترس قرار می‌دهد.

> **چرا statusLine و نه یک تایمر؟** `rate_limits` رسمی فقط به یک فرمان statusLine
> تحویل داده می‌شود. یک تایمر پس‌زمینه هیچ راهی برای گرفتن آن‌ها ندارد، پس فقط
> می‌تواند مصرف را از روی شمارش توکن‌های محلی *تخمین* بزند. برای مطابقت با عددی که
> Claude Code نشان می‌دهد، افزونه روی statusLine سوار می‌شود — که در هر رندر
> تازه می‌شود (زیر یک ثانیه وقتی فعال هستید)، بیشتر از یک نظرسنجی ۱۰ ثانیه‌ای.

## ابزارها

<div dir="ltr">

| ابزار | ورودی | مخرب؟ |
| --- | --- | --- |
| `check_usage` | ندارد | خیر |
| `check_data_status` | ندارد | خیر |

</div>

### `check_usage`

بدون ورودی. مصرف فعلی را به‌صورت متن برمی‌گرداند:

<div dir="ltr">

```json
{
  "usage_percent": 86,
  "resets_at": "2026-07-06T18:00:00.000Z",
  "seven_day_percent": 80,
  "seven_day_resets_at": "2026-07-11T00:00:00.000Z",
  "stale": false
}
```

</div>

`usage_percent` / `resets_at` مربوط به بازه‌ی **۵ ساعته** است؛ `seven_day_*`
مربوط به بازه‌ی **هفتگی** است. `stale` وقتی `true` می‌شود که ذخیره‌ی داده قدیمی‌تر
از ۱۰ دقیقه باشد.

به‌جای شکست خاموش، خطاهای ساختاریافته برمی‌گرداند:

- `{"error": "statusline_not_configured"}` — هوک statusLine هنوز وصل نشده است
  (به [پیکربندی هوک statusLine](#پیکربندی-هوک-statusline) مراجعه کنید).
- `{"error": "rate_limits_unavailable"}` — یک ذخیره وجود دارد اما Claude Code
  `rate_limits` را گزارش نمی‌کند (به نسخه ۲.۱.۸۰ به بالا روی پلن Pro/Max نیاز دارد).

### `check_data_status`

بدون ورودی. وقتی یک ذخیره موجود و تازه باشد (در ۱۰ دقیقه‌ی اخیر به‌روزرسانی شده)
`{"authenticated": true}` و در غیر این‌صورت `{"authenticated": false}`
برمی‌گرداند. برای یک بررسی سریع سلامت پیش از فراخوانی `check_usage` مفید است.

### پیش‌نیازها

- [Node.js](https://nodejs.org/) نسخه ۱۸ به بالا

  **Pro** یا **Max** — نسخه‌های قدیمی‌تر و پلن‌های دیگر `rate_limits` را در


### پیکربندی هوک statusLine

سرور MCP فقط ذخیره‌ی داده را *می‌خواند*؛ این هوک statusLine است که آن را تازه نگه
می‌دارد. از آنجا که Claude Code فقط `rate_limits` رسمی را به یک فرمان statusLine
تحویل می‌دهد — و statusLine یک مؤلفه‌ی قابل‌ارائه توسط افزونه نیست — این را یک‌بار
به `~/.claude/settings.json` خود اضافه کنید:

<div dir="ltr">

```json
{
  "statusLine": {
    "type": "command",
    "command": "node \"C:/path/to/claude-usage-plugin/capture-statusline.js\""
  }
}
```

</div>

روی ویندوز در مسیر از **اسلش رو به جلو** استفاده کنید — Claude Code این رشته را با
تقسیم‌کلمه به سبک POSIX تجزیه می‌کند، جایی که بک‌اسلش‌ها خراب می‌شوند.

> **از قبل یک statusLine دارید؟** آن را جایگزین نکنید — stdin را به هر دو فرمان
> زنجیر کنید، یا فراخوانی `captureStatusline()` از `official.js` را در اسکریپت
> موجود خود ادغام کنید.

Claude Code را دوباره راه‌اندازی کنید، یک پیام بفرستید تا هوک اجرا شود، سپس
`check_usage` اعداد رسمی زنده‌ی شما را گزارش می‌دهد.

## استفاده

### نمونه یک نشست

یک نشست معمول پس از نصب افزونه و وصل‌شدن هوک statusLine:

1. **«آیا داده‌ی مصرف من هنوز موجود است؟»** → فراخوانی `check_data_status`، که پس
   از حداقل یک‌بار اجرا شدن هوک `{"authenticated": true}` برمی‌گرداند
2. **«چقدر از مصرفم را استفاده کرده‌ام؟»** → فراخوانی `check_usage`، که درصدهای ۵
   ساعته و هفتگی به‌همراه زمان‌های بازنشانی‌شان را برمی‌گرداند

چون نه ورودی‌ای هست و نه محدودیتی، کل سطح کار همین است: بپرسید، و افزونه گزارش
می‌دهد.

## یادداشت‌های طراحی

- **رسمی، نه تخمینی.** اعداد همان `rate_limits` خود Claude Code هستند.
- **بدون محدودیت.** افزونه فقط ثبت و گزارش می‌کند — هیچ‌چیز بلاک نمی‌شود.
- **حداقل توکن.** دو ابزار بدون ورودی که متن برمی‌گردانند؛ هوک statusLine فقط در
  سطح harness است و چیزی به کانتکست مدل اضافه نمی‌کند.
- **بدون تماس شبکه.** `index.js` فقط یک فایل JSON محلی را می‌خواند.
- **نوشتن اتمیک.** ذخیره در یک فایل موقت یکتا به ازای pid نوشته و سپس به جای خود
  تغییرنام داده می‌شود، پس یک خواندن هم‌زمان هرگز فایل نیمه‌نوشته نمی‌بیند.
- **هرگز نوار وضعیت را خالی نمی‌کند.** خطاهای ذخیره و قالب‌بندی بلعیده می‌شوند، پس
  هوک نمی‌تواند پرامپت شما را خراب کند.
